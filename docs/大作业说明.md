# PCB 元件布局优化 — 大作业

## 1. 问题描述

在 PCB（印制电路板）设计中，元件布局（Placement）决定了每个电子元件在电路板上的物理位置。好的布局能够缩短走线长度、减少信号延迟、降低布线难度。

本作业要求你编写程序，对给定的 PCB 电路数据，**调整元件的位置，使得总线长（HPWL）最小化**。

### 优化目标：HPWL

采用 **半周长线长（Half-Perimeter Wirelength, HPWL）** 作为优化目标：

$$
\text{HPWL} = \sum_{\text{net}} \left[ \max_i(x_i) - \min_i(x_i) + \max_i(y_i) - \min_i(y_i) \right]
$$

其中 $(x_i, y_i)$ 是该 net 中第 $i$ 个引脚的绝对坐标。**HPWL 越小，布局质量越好。**

引脚绝对坐标的计算方式：

$$
x_{\text{pin}} = x_{\text{comp}} + \frac{w_{\text{comp}}}{2} + dx
$$

$$
y_{\text{pin}} = y_{\text{comp}} + \frac{h_{\text{comp}}}{2} + dy
$$

其中 $(x_{\text{comp}}, y_{\text{comp}})$ 是元件左下角坐标，$w, h$ 是元件的宽和高，$(dx, dy)$ 是引脚相对于元件中心的偏移量。

### 约束条件

- **最小间距**：任意两个元件的矩形之间必须保持至少 **2 个单位** 的间距。

  对于元件 A（位置 $(x_A, y_A)$，尺寸 $w_A \times h_A$）和元件 B（位置 $(x_B, y_B)$，尺寸 $w_B \times h_B$），定义两个方向上的间距：

  $$\text{gap}_x = \max(x_A,\ x_B) - \min(x_A + w_A,\ x_B + w_B)$$

  $$\text{gap}_y = \max(y_A,\ y_B) - \min(y_A + h_A,\ y_B + h_B)$$

  其中 $\text{gap}_x > 0$ 表示两矩形在 x 方向上分离，$\text{gap}_x < 0$ 表示在 x 方向上投影重叠。

  **间距判定规则**：两个元件满足最小间距要求，当且仅当 $\text{gap}_x \geq 2$ **或** $\text{gap}_y \geq 2$。换言之，两个矩形只要在 x 或 y 任一方向上拉开了至少 2 个单位，即为合法。

- **不越界**：所有元件应放置在给定的画布范围内。

## 2. 数据格式（Bookshelf 格式）

每个数据集包含 3 个文件：

| 文件 | 用途 |
|------|------|
| `*.nodes` | 元件名称、尺寸（宽、高） |
| `*.nets` | 网络连接关系及引脚偏移量 |
| `*.pl` | 元件的放置位置 |

### 2.1 `.nodes` — 元件定义

```
UCLA nodes 1.0

NumNodes : 115
NumTerminals : 0

R96_3 5 10            # 元件名 宽度 高度
R52_3 10 5
U2_3 71 71            # 大型 BGA 芯片
C20_3 17 10           # 中型 IC
```

每行格式：`<名称> <宽度> <高度>`

### 2.2 `.nets` — 网络与引脚

```
UCLA nets 1.0

NumNets : 199
NumPins : 334

NetDegree : 2 NET_12V_HVDD_DVDD_ASIC1    # 网络名，包含 2 个引脚
    R59_3 I : -4 0                        # 元件 R59_3 的引脚，偏移 (-4, 0)
    R87_3 I : 0 -4                        # 元件 R87_3 的引脚，偏移 (0, -4)
NetDegree : 7 NET_3V3_CTRL_HVDD_DVDD_PL0_3   # 7 个引脚
    R37_3 I : 4 0
    R38_3 I : 4 0
    ...
    U2_3 I : -34 -18                     # BGA 芯片上的引脚
```

每行格式：`<元件名> I : <dx> <dy>`，其中 $(dx, dy)$ 是相对于**元件中心**的偏移量。

### 2.3 `.pl` — 元件位置

```
UCLA pl 1.0

R96_3 2500 1773 : W       # 元件 R96_3，左下角在 (2500, 1773)，朝向 W
R52_3 3762 2031 : N       # 朝向 N（North，默认朝向）
U2_3 3693 2056 : E        # BGA 芯片，朝向 E
```

每行格式：`<名称> <x> <y> : <朝向>`

> **注意**：$(x, y)$ 是元件矩形**左下角**的坐标。元件中心为 $(x + w/2,\ y + h/2)$。朝向可为 N/E/S/W，表示元件的旋转方向。

## 3. 示例：small-5 数据集

以 `small-5` 为例说明数据结构和优化任务。该数据集包含：

| 统计项 | 数值 |
|--------|------|
| 元件总数 | 115 |
| 网络（Nets）数 | 199 |
| 引脚（Pins）总数 | 334 |

### 元件类型分布

- **小型无源元件**（电阻/电容）：尺寸多为 5×10 或 10×5，数量最多（如 R96_3, R63_3, C18_3）
- **中型 IC**：尺寸为 17×10 或 10×18（如 C20_3, C35_3, R27_3）
- **大型 BGA 芯片**：3 个 71×71 的元件（U1_3、U2_3、U3_3），是电路核心

### 优化前后对比

下方为 small-5 优化前后的布局对比（同一网表，仅元件位置不同）：

![small-5 优化前后对比](results/small-5_comparison.png)

| | 优化前 | 优化后（AutoDMP） |
|---|--------|------------------|
| 元件数 | 115 | 115 |
| 网络数 | 199 | 199 |

可以看到，优化前元件分散在画布各处，连线跨越很远；优化后元件围绕 3 个 BGA 芯片紧密排列，连线大幅缩短。

### HPWL 计算示例

以 `NET_3V3_CTRL_HVDD_DVDD_PL0_3` 为例（7 个引脚的 net）：

```
NetDegree : 7 NET_3V3_CTRL_HVDD_DVDD_PL0_3
    R37_3 I : 4 0
    R38_3 I : 4 0
    R40_3 I : -4 0
    R56_3 I : 0 4
    C19_3 I : -4 0
    C20_3 I : -7 0
    U2_3 I : -34 -18
```

假设优化前 U2_3 位于 (3693, 2056)，宽高 71×71，则 U2_3 上该引脚的绝对坐标为：

$$x = 3693 + 71/2 + (-34) = 3693 + 35.5 - 34 = 3694.5$$

$$y = 2056 + 71/2 + (-18) = 2056 + 35.5 - 18 = 2073.5$$

对所有 7 个引脚计算绝对坐标后，该 net 对 HPWL 的贡献为：

$$\Delta\text{HPWL} = (\max x_i - \min x_i) + (\max y_i - \min y_i)$$

## 5. 方法要求

**不限制优化方法。** 你可以使用任何你选择的算法，包括但不限于：

- 模拟退火（Simulated Annealing）
- 遗传算法（Genetic Algorithm）
- 力导向法（Force-Directed）
- 贪心策略 / 局部搜索
- 其他方法

你可以同时使用多种方法进行对比实验。
